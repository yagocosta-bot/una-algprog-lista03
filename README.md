# una-algprog-lista03
Este repositório contém uma série de exercícios introdutórios de lógica de programação. Cada atividade apresenta um contexto real, uma tarefa objetiva e, quando necessário, fórmulas para apoiar os cálculos.  

Escolher entre inteiro e real é importante para evitar erros de cálculo:

Inteiro: usado para valores que não precisam de casas decimais (ex.: contagens).
Real: usado quando o valor pode ter frações (ex.: dinheiro, temperaturas, medições).

Usar o tipo errado pode causar arredondamentos indesejados, perda de precisão ou resultados incorretos, especialmente em cálculos financeiros e científicos.

Exercício 1 — O Fechamento do Caixa  
Contexto:  
Um pequeno mercado deseja automatizar o processo de fechamento do caixa.  
Tarefa:  
Criar um algoritmo que leia:  

o nome do produto  
o preço unitário  
a quantidade comprada  

O programa deve calcular o valor total e exibir a frase:  

“O produto [produto] custou um total de R$ [valorTotal].”  

Exercício 2 — Conversor de Temperatura (Fahrenheit)  
Contexto:  
Você está desenvolvendo um software para uma estação meteorológica americana. O sistema precisa converter temperaturas de Celsius para Fahrenheit.  
Tarefa:  
Ler uma temperatura em graus Celsius e convertê-la para Fahrenheit utilizando a fórmula:  
F = (C × 1.8) + 32  

Exercício 3 — Média Ponderada  
Contexto:  
Uma faculdade utiliza pesos diferentes para duas provas:  

Prova A → peso 4  
Prova B → peso 6  

Tarefa:  
Ler as duas notas do aluno e calcular a média ponderada, usando:  
Média = (NotaA × 4 + NotaB × 6) / 10  

Exercício 4 — Consumo do Automóvel  
Contexto:  
Um aplicativo de caronas deseja calcular o consumo médio de combustível de um veículo.  
Tarefa:  
Ler a:  

distância percorrida (km)  
quantidade de combustível consumido (litros)  

O sistema deve exibir o consumo médio (km/l) do carro.  

Exercício 5 — Cálculo de Desconto Profissional  
Contexto:  
Um e‑commerce oferece descontos especiais em feriados.  
Tarefa:  
Ler:  

o valor original do produto  
o percentual de desconto (ex.: 15)  

O programa deve calcular:  

O valor do desconto em reais  
O novo preço final após o desconto
